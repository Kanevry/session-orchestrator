Loop engineering for AI coding agents

View on GitHub Install

Why this design

Four things it does that ad-hoc agent sessions don't.

Verification gates, not vibes

Every close runs a Full Gate — typecheck, test, lint — before anything ships. No completion claims without fresh evidence.

Wave orchestration

research → plan → wave-execute → close, in five typed waves. Parallel subagents with declared file-scope enforcement and a quality gate between every wave.

Memory that compounds

Sessions write learnings; a reconciliation engine turns confidence-scored patterns into rule proposals you review — never auto-applied.

Runs where you work

The same skills and commands on Claude Code, Codex CLI, Cursor IDE, and Pi — with platform-adapted hooks and enforcement.

Install

One command to get started, per harness.

Claude Code

/plugin marketplace add Kanevry/session-orchestrator
/plugin install session-orchestrator@kanevry

Plugin marketplace — run both lines inside Claude Code.

Codex CLI

git clone https://github.com/Kanevry/session-orchestrator.git ~/Projects/session-orchestrator
cd ~/Projects/session-orchestrator && npm install
node scripts/codex-install.mjs

Registers the plugin in your local Codex plugin catalog. Details: Codex setup guide.

Cursor IDE

Rules-based install (.mdc), sequential execution — see the Cursor setup guide.

Pi

git clone https://github.com/Kanevry/session-orchestrator.git ~/Projects/session-orchestrator
cd ~/Projects/session-orchestrator && npm install
node scripts/pi-install.mjs /path/to/your/project --settings-only

pi install npm:session-orchestrator — on npm since v3.16.0.
